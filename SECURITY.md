# Security Policy

## Supported versions

Security fixes are applied to the latest branch under active development.

## Reporting a vulnerability

Please do not open public issues for security problems.

1. Open a private report using GitHub Security Advisories (preferred).
2. If that is not available, contact the maintainers privately through the project contact channel listed in the app/repository profile.
3. Include reproduction steps, impact, and affected files/endpoints.

You can expect an initial acknowledgment within 5 business days.

## Scope focus for this project

When reporting, prioritize:

- Data leakage (PII, local data export, logs, crash reports)
- Proxy authentication and rate-limit bypasses
- Prompt/debug endpoint exposure
- Supply-chain risks in dependencies
