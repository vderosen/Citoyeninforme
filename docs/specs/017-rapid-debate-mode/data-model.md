# Data Model: Rapid Debate Context

**Feature**: 017-rapid-debate-context
**Date**: 2026-02-20

## Entities

### DebateTurn

A single exchange in the debate. Generated by the LLM as a structured JSON response.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| id | string | yes | Unique identifier, generated client-side (e.g., `"turn-{timestamp}-{index}"`) |
| statement | string | yes | The AI's argument or provocative question (2-4 sentences) |
| options | DebateOption[] | yes | 2-4 response options (empty array for conclusion turns) |
| selectedOptionId | string or null | no | The user's selected option ID. Null if this is the current (unanswered) turn |
| themeId | string or null | no | The election theme this turn relates to (matches `Theme.id` from election dataset) |
| sources | Source[] | no | Source citations for claims made in the statement |
| isConclusion | boolean | yes | Whether this is the final summary turn |
| summary | DebateSummary or null | no | Present only on conclusion turns |
| timestamp | string | yes | ISO 8601 creation timestamp |

**Validation Rules**:
- `options` must have 2-4 items when `isConclusion` is false
- `options` must be empty when `isConclusion` is true
- `summary` must be present when `isConclusion` is true
- `statement` must not be empty

**State Transitions**:
```
Created (selectedOptionId=null) → Answered (selectedOptionId="{id}")
```
A turn is immutable once answered. No undo.

---

### DebateOption

A single selectable response within a debate turn.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| id | string | yes | Short identifier: "a", "b", "c", or "d" |
| text | string | yes | Display text for the option (1-2 sentences) |

**Validation Rules**:
- `id` must be one of "a", "b", "c", "d"
- `text` must not be empty

---

### Source

A citation for a claim made in a debate turn statement. Reuses the existing `SourceReference` pattern from the election dataset.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| title | string | yes | Human-readable source title |
| url | string | no | URL to the source document |

---

### DebateSummary

The structured conclusion generated at the end of a debate.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| themesExplored | string[] | yes | List of theme IDs discussed during the debate |
| keyInsight | string | yes | A paragraph summarizing the user's argumentation patterns and tendencies |
| candidateProximity | CandidateProximityEntry[] | no | Factual proximity to candidates, only when justified |

**Validation Rules**:
- `themesExplored` must contain at least one theme ID
- `keyInsight` must not be empty
- `candidateProximity` may be empty or absent when no justified match exists

---

### CandidateProximityEntry

A single factual observation linking the user's debate positions to a candidate's documented positions.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| candidateId | string | yes | References `Candidate.id` from election dataset |
| reason | string | yes | Specific justification text explaining the proximity |

**Validation Rules**:
- `candidateId` must reference an existing candidate in the election dataset
- `reason` must reference specific positions defended during the debate AND documented candidate positions

---

## In-Memory State (Not Persisted)

### DebateState (within assistant store)

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| debateTurns | DebateTurn[] | [] | Ordered list of all turns in the current debate |
| isDebateActive | boolean | false | Whether a debate session is in progress |
| isGeneratingTurn | boolean | false | Whether the system is waiting for the LLM to generate the next turn |
| debateStartThemeId | string or null | null | Theme chosen by user at start (no-survey path) |

**Lifecycle**:
```
Idle (isDebateActive=false, debateTurns=[])
  → startDebate(themeId?)
Active (isDebateActive=true, turns accumulate)
  → selectOption(turnId, optionId) → generates next turn
  → endDebate() or conclusion turn received
Concluded (last turn has isConclusion=true)
  → resetDebate() → back to Idle
```

**Not persisted**: These fields are excluded from Zustand's `partialize` function. On app restart, the debate state resets to defaults.

---

## Relationships to Existing Entities

```
DebateTurn.themeId → Theme.id (election dataset)
DebateSummary.themesExplored[] → Theme.id (election dataset)
CandidateProximityEntry.candidateId → Candidate.id (election dataset)
DebateState uses UserProfile.contradictions (survey store) for auto-start
DebateState uses UserProfile.themeScores (survey store) for context
```

## JSON Response Schema (LLM Output)

The LLM is instructed to produce responses matching this schema:

**Normal Turn**:
```json
{
  "statement": "string (2-4 sentences)",
  "options": [
    { "id": "a", "text": "string" },
    { "id": "b", "text": "string" }
  ],
  "themeId": "string | null",
  "sources": [
    { "title": "string", "url": "string" }
  ]
}
```

**Conclusion Turn**:
```json
{
  "statement": "string (summary paragraph)",
  "options": [],
  "isConclusion": true,
  "summary": {
    "themesExplored": ["string"],
    "keyInsight": "string",
    "candidateProximity": [
      { "candidateId": "string", "reason": "string" }
    ]
  }
}
```
